# Test Failures due possible code bugs

## IdentityProvider.Tests.csproj - ExecuteAsync_InvalidJsonMessage_SkipsNullEvent
- **Confidence**: High
- **Test File**: IdentityProvider\IdentityProviderTests\Consumers\UserPromotedConsumerTests.cs
- **Bug Location**: IdentityProvider\Consumers\UserPromotedConsumer.cs@40

### Analysis
The test is skipped because the production code (UserPromotedConsumer.ExecuteAsync) directly instantiates a Kafka consumer on line 40 using 'new ConsumerBuilder<string, string>(config).Build()', which creates a real Kafka consumer that cannot be mocked. This architectural issue makes it impossible to unit test the null event handling scenario (line 54) without either: (1) refactoring the production code to inject the Kafka consumer as a dependency, or (2) using integration testing with a real Kafka broker. The test comments (lines 158-160 in the test file) explicitly document this limitation. This is not a bug in the test itself - the test correctly identifies that it cannot be run as a unit test with the current architecture.

### Suggested Fix
To make this testable, refactor UserPromotedConsumer to accept an abstraction for the Kafka consumer. Options include:

1. Create an interface IKafkaConsumerFactory:
```csharp
public interface IKafkaConsumerFactory<TKey, TValue>
{
    IConsumer<TKey, TValue> CreateConsumer(ConsumerConfig config);
}
```

2. Inject this factory in the constructor:
```csharp
private readonly IKafkaConsumerFactory<string, string> _consumerFactory;

public UserPromotedConsumer(
    IServiceScopeFactory scopeFactory,
    IOptions<KafkaOptions> options,
    ILogger<UserPromotedConsumer> logger,
    IKafkaConsumerFactory<string, string> consumerFactory)
{
    _scopeFactory = scopeFactory;
    _options = options;
    _logger = logger;
    _consumerFactory = consumerFactory;
}
```

3. Replace line 40 in ExecuteAsync:
```csharp
using var consumer = _consumerFactory.CreateConsumer(config);
```

With this change, the test can mock the consumer factory and return a mocked IConsumer<string, string> that simulates invalid JSON deserialization scenarios.
